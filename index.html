The Go Programming Language
Go is an open source programming language that is statically typed and compiled. It has many features including memory allocation, garbage collection, and built-in support for concurrent programming.

The development of Go began because of problems with:

  • network systems
  • multicore-processors
  • massive computation clusters

Robert Griesemer, Rob Pike, and Ken Thompson began to develop Go in 2007 and it was first released in 2009. It was created to handle the large scale of programming at Google.

It is now used in part for services and projects developed by Google, Dropbox, MongoDB, Soundcloud, Uber, and many others.

The code below creates the Adventurer struct. Note that types are required and postscripted.

      	type Adventurer struct {
      		hp, mp int
      	}

Go allows for blank returns. The return type of the NewAdventurer function returns an Adventurer pointer named tom. This automatically initializes tom to the specified type. The blank return will return the variable named as the return value.

      	func NewAdventurer() (tom *Adventurer){
      		tom = &Adventurer{hp:10,mp:5}
      		return
      	}

Similar to prototyping in Javascript, the following code attaches the GetHP() and Heal() function to an object. As a result, objects can be simulated in Go.

When the var keyword is used in function declarations, the type can be omitted. Go is statically typed, so it is still necessary to keep track of variable types.

      	func (tom *Adventurer) GetHP() int{
	      	return tom.hp
		}

		func (tom *Adventurer) Heal() int{
	    	var healed = rand.Intn(5)
	    	(*tom).hp += healed
	    	return healed
		}

Similarly to var, the := assignment can be used for implicit type declaration for variables defined within functions.

	func (tom *Adventurer) Damage() int{
		damaged := rand.Intn(5)
		(*tom).hp -= damaged
		return damaged
	}

Parentheses are not necessary for control statements. The code below also shows how Go allows for multiple operations in if and for statements when separated by semicolons. The else keyword must also go on the same line as the ending if bracket.

	func (tom *Adventurer) DoSomething(){
		var change int
		fmt.Printf("You pick something off the floor. You dust it off
            and eat it.\n")
		time.Sleep(2 * time.Second)
		if chance := rand.Float32() ; chance < 0.5 {
			change = tom.Damage()
			fmt.Printf("It's poison. You lose %d hp.\n",change)
		} else {
			change = tom.Heal()
			fmt.Printf("It's a cookie. Moldy, but good. It heals you for
                %d hp.\n",change)
		}
	}

Nows a time to note other quirks and conventions of Go.

  • There are no while loops. Everything is a for loop.
  • Function names are capitalized and camelcased.
  • Go enforces good practices. Programmers must use all variables and libraries.
  • Go's goroutines are a lightweight method of executing concurrent programs. Learn more here.
